# 通用 Agent 范式：谁相信，谁质疑

**AI 行业正在收敛于一个强大的理念：强大的基础模型 + 通用 Agent 运行时 + 可组合的工具，可以取代大多数特定领域的 AI Agent。** Anthropic 已将这一理念明确化——把 Claude Code SDK 重命名为"Claude Agent SDK"，并推出 Agent Skills（可组合的 markdown 文件，将通用 Agent 变成领域专家）。但全貌比任何一家公司的定位都更微妙。Andrej Karpathy 认为仍需要"厚应用层"。Yann LeCun 称通用智能"完全是扯淡"。VC 们在押注水平平台还是垂直专家之间分裂。与此同时，围绕 Claude Code 已经涌现出一个丰富的高级工作流生态系统，将一个编程工具变成了覆盖研究、写作、金融、科学和组织管理的通用自动化平台。

---

## 第一部分：行业领袖眼中的"模型 + Agent + 工具"范式

### Anthropic 的明确押注：一个 Agent 运行时统治一切

Anthropic 是通用 Agent 范式最清晰、最明确的支持者。他们的产品演进讲述了整个故事：Claude Code 以编程工具起步，演变为通用自动化平台，最终正式更名以反映这一愿景。

**Alex Albert**（Anthropic）在 2025 年 9 月 29 日表示："我们已经将 Claude Code SDK 重新命名为 Claude Agent SDK，以反映它是构建任何通用 Agent 的最佳方式，而不仅仅是编程 Agent。" 配套的工程博客更加直白："在过去几个月中，Claude Code 已经远远超越了编程工具的范畴。在 Anthropic 内部，我们一直用它做深度研究、视频制作和笔记整理等无数非编程应用。事实上，它已经开始驱动我们几乎所有主要的 Agent 循环。"

设计哲学的核心是一个看似简单的理念：**给 Agent 一台计算机。** 与其为每个领域构建专门的工具 API，不如给 Claude bash 命令、文件操作和搜索——和人类程序员使用的原语完全一样。正如工程团队所写："通过赋予它运行 bash 命令、编辑文件、创建文件和搜索文件的工具，Claude 可以读取 CSV、搜索网页、构建可视化、解读指标，以及执行各种数字工作——简而言之，用一台计算机创建通用 Agent。"

2025 年 10 月推出的 **Agent Skills** 使组合模式更加明确。Skills 是将领域专业知识打包为可组合资源的 markdown 文件："不再为每个用例构建碎片化、定制设计的 Agent，任何人现在都可以用可组合的能力来专业化自己的 Agent。" 这就是范式的精华：通用 Agent 底座加上任何人都能编写的插件式专业化。

Anthropic 的基础性文章《构建有效的 Agent》（2024 年 12 月，Erik Schluntz 和 Barry Zhang 撰写）奠定了思想基础："始终如一地，最成功的实现并没有使用复杂的框架或专门的库。相反，它们使用简单、可组合的模式来构建。" 信息很明确——简单性和可组合性胜过专业化的复杂性。

### Karpathy 的中间路线："Cursor for X"应用层

Andrej Karpathy，可能是 AI 工程领域最具影响力的独立声音，采取了审慎的中间立场。他肯定通用 Agent 基础，同时主张将持续存在一个厚实的应用层。

他的"LLM 操作系统"愿景（2023 年 9 月）极具前瞻性："一个更完整的图景正在浮现——LLM 不是聊天机器人，而是新操作系统的内核进程。" 到 2025 年 6 月，这演变为 YC AI Startup School 上的 **"Software 3.0"** 框架："神经网络通过大型库变得可编程。这是一种新型计算机。在我看来，值得将其命名为 Software 3.0。" 他宣称 "Software 3.0 正在吞噬 1.0/2.0"，并预测"大量软件将被重写。"

然而 Karpathy 拒绝接受通用 Agent 消除专业产品需求的结论。在他 2025 年 12 月的年度回顾中，他写道："我怀疑 LLM 实验室会趋向于培养出一般能力的大学毕业生，但 LLM 应用会通过提供私有数据、传感器和执行器以及反馈循环，来组织、微调并实际驱动这些'毕业生'成为特定垂直领域的专业人士。" "Cursor for X" 模式——做上下文工程、编排多次 LLM 调用、提供领域特定 GUI、提供**"自主权滑块"**的垂直应用——代表着一个厚实且可防御的应用层。

值得注意的是，Karpathy 将 Claude Code 认定为"LLM Agent 的首个令人信服的演示——以循环方式将工具使用和推理串联起来，进行延展性问题解决。" 他批评 OpenAI 过于关注云端 Agent 部署而非本地运行："我认为 OpenAI 搞错了，因为他们将早期的 Codex/Agent 精力放在了从 ChatGPT 编排的云端容器部署上，而不是简单的 localhost。"

他的关键概念贡献是 **"参差不齐的智能"（jagged intelligence）** ——观察到前沿 LLM 能解决复杂数学却在极简单任务上犯错。这种固有的不均匀性是为什么他将当前时刻定义为 **"Agent 的十年"** 而非 "Agent 的一年"——完全自主需要谨慎、渐进地扩展自主权滑块。

### OpenAI 的趋同演化：通用之中的专业化

OpenAI 的路径揭示了富有成效的张力。他们创建了 GPT-5-Codex，一个"专为 Codex CLI 打造"且明确"不是通用模型"的模型——与 Anthropic 的通用主义立场形成直接对比。然而他们的年终开发者回顾承认了趋同："到 2025 年中后期，这些能力被整合进 GPT-5 系列……不再与通用模型分离，而是在通用模型内部专业化。"

Sam Altman 在 2025 年 1 月的"反思"博文中设定了战略框架："我们现在有信心知道如何构建我们传统意义上理解的 AGI。我们相信在 2025 年，我们可能会看到第一批 AI Agent'加入劳动力'，并实质性地改变公司的产出。" 他后来的"温和奇点"博文延伸了时间线："2025 年见证了能做真正认知工作的 Agent 的到来……2035 年的任何人都应该能够调动相当于 2025 年所有人的智力资源。"

在 Sequoia AI Ascent 2025 上，Altman 捕捉到了演进模式："从助手到 Agent 到完整应用的进化将感觉是连续的，而非离散的。" OpenAI 的 Codex 也通过 MCP 拥抱了可组合性，允许 Codex CLI 作为 MCP 服务器供其他 Agent 调用——与 Anthropic 倡导的互操作范式如出一辙。

### Google DeepMind 将 Agent 视为通向 AGI 的路径

Demis Hassabis 将 Agent 能力定位为迈向通用人工智能的中间步骤。在 Gemini 2.0 发布会（2024 年 12 月）上，他表示："你实际上需要一个系统不仅仅给你信息，而是能够去为你完成任务……如果我们思考通向 AGI 的道路，那么显然你需要一个能推理、分解问题并在世界中执行行动的系统。" Google 的方法使用 Gemini 作为通用基础，驱动多样化应用——Co-Scientist 做研究、Astra 做多模态理解，以及各种企业 Agent——隐含地支持"一个模型，多个应用"的范式。

### Nadella 的激进预测：SaaS 坍缩进 Agent 层

也许最具挑衅性的声明来自 **Satya Nadella** 在 2024 年 12 月的 BG2 播客上："商业应用存在这个概念——在 Agent 时代它们可能都会坍缩，因为你想想，它们本质上就是 CRUD 数据库加上一堆业务逻辑。业务逻辑全都会转移到这些 Agent 上……所有逻辑都会在 AI 层。一旦 AI 层成为所有逻辑所在的地方，人们就会开始替换后端。" 他更进一步："嘿，我为什么需要 Excel？……我们会非常积极地尝试把一切都坍缩掉。" 这将通用 Agent 定位为不仅仅是工具，而是整个企业软件品类的替代者。

### 最强反对声：LeCun、Baudiš，以及专业化的理由

**Yann LeCun**（Meta）代表了哲学上的对立极。2025 年 12 月，他宣称："不存在通用智能这种东西。这个概念完全没有意义。" 他的数学论证是：人类大脑大约可以表示 **2^(3.2×10^15)** 个函数——与所有可能的视觉函数相比，这是"一个极小的数字"。即使人类也是"极度专业化的。" 在 CES 2025 上，他预测 AI Agent 将是"训练用于特定任务的机器人，而非真正能从零开始执行活动的智能助手"，并警告演示与部署之间的鸿沟："你可以看到令人印象深刻的演示。但当真正要部署一个足够可靠的系统时……差距很大。"

**Petr Baudiš**（Rossum CTO）撰写了《通用 AI Agent 的不合理炒作》，指出三个失败模式：**领域专业知识**（"在看似令人印象深刻的表面知识薄纱之下，往往什么都没有"）、**领域整合**（专家经验需要与目的相关的操作限制）、以及**质量控制**（"给一个自信但资浅的人不受约束的权力"）。

IBM 研究人员强化了这种怀疑。**Marina Danilevsky** 警告："Agent 往往非常低效，因为人类是很差的沟通者。" **Maryam Ashoori** 更直接："你组织中的一切都应该用 Agent 流程编排吗？不，这行不通。"

学术研究增添了细微差别。一篇 2025 年的 arXiv 论文认为，"专业化的专家而非扩大单体模型，能产生更便宜、更易调试、更易部署的系统。" NeurIPS 2024 的 AgentBoard 基准测试发现，即使顶级 LLM 在多领域 Agent 任务上也表现吃力，突显了当前通用主义的局限性。

### 正在形成的共识：可组合专业化的混合编排

最精通的声音趋同于混合模型。**Harrison Chase**（LangChain CEO）在 Sequoia 播客上明确表述："当 Agent 搞砸时，是因为它们没有正确的上下文；当它们成功时，是因为它们有正确的上下文。" 他的洞察重新定义了整场辩论：区别不在于通用 vs. 专业化——而在于**上下文工程**的质量。

**Simon Willison**——他在 2024 年末预测"大量资金将在追逐几个定义不清的梦想中亏损"——完全逆转了自己的立场："事后看来，Claude Code 这个名字取得不好。它不纯粹是编程工具：它是通用计算机自动化工具。任何你能通过在计算机上敲命令实现的事情，现在都可以被 Claude Code 自动化。最好把它描述为通用 Agent。" 他后来补充："我一直在说 Claude Code 是一个'伪装成开发者工具的通用 Agent'。"

一个实际案例来自 **Boris Renski**（Apelogic），他构建了一个专业化的数据 Agent，结果"其实效果不太好"——脆弱、缓慢、经常出错。他转向 Claude 加 MCP 连接器，获得了显著更好的结果："通用 Agent 已经能处理专业化 Agent 约 95% 的工作，而且随着前沿模型的改进，这个比例还在攀升。"

VC 们仍然分裂。**a16z** 大力投资垂直 AI（"取代通用工具的垂直特定 AI Agent"），约 40% 的 AI 投资组合在医疗健康领域。**Bessemer** 认为垂直 AI "在争夺劳动力预算，而非 IT 预算。" 但最精通的投资者描述的是一种 **"通用编排者 + 专业化工人"** 架构——一个中心 Agent 将工作拆分为任务，分配给领域专家。

中文 AI 社区得出了类似结论：**"通用底座+垂直模块"** 是 2025 年的主流发展方向。中文分析师观察到通用 Agent "尚不成熟"，而垂直 Agent "正在达到可用水平"——但将垂直能力插到通用底座上的架构已被广泛接受。IDC 预测中国 AI Agent 市场将从 2025 年的 **554 亿元增长到 2028 年的 8520 亿元**，复合年增长率 **72.7%**。

LangChain 对生产级 Agent 的分析从经验层面证实了这一模式："像 Claude Code 和 Manus 这样的通用 Agent 使用的工具数量极少。Claude Code 用了大约十几个，Manus 用了不到 20 个。关键洞察是给 Agent 访问计算机的权限。有了 bash 和文件系统工具，Agent 就能像人类一样执行操作，而不需要为每个任务绑定专门的工具。"
