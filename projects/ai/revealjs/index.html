质疑者

LeCun 是最强反对声。他从数学角度论证即使人类也是"极度专业化的"。 他警告演示与部署之间的鸿沟很大。

反对派的三个核心论点

领域专业知识缺失

"在看似令人印象深刻的表面知识薄纱之下，往往什么都没有" — Baudiš

领域整合困难

专家经验需要与目的相关的操作限制，通用 Agent 缺乏这种约束

质量控制风险

"给一个自信但资浅的人不受约束的权力" — IBM: "Agent 往往非常低效"

Baudiš 指出三个失败模式：领域专业知识缺失、整合困难、质量控制风险。 IBM 研究人员警告 Agent 非常低效，不是所有流程都适合 Agent 编排。

那么，谁对了？

答案比两个阵营都更微妙

过渡到共识部分。答案不是简单的通用或专业化——而是一种新的混合架构。

范式是真实的
但实现是分层的

✓ 方向性论点成立 — 通用 Agent 正在取代许多专业化系统

✓ 但不是"纯通用"也不是"纯专业化"
